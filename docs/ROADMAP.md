# Development Roadmap

## Phase 1: Foundation

- **Goals**: Understand how to create module system:
  - Build a module
  - Build CLI interface
  - Plan your interfaces to create abstractions

### Features

- [ ] Basic task management module (create, read, update, delete tasks)
- [ ] CLI interface to interact with the task module
- [ ] SQLite and in-memory storage
- [ ] Plan module system

## Phase 2:

- **Goals**: Create abstractions and a modular system:
  - Build `core` for module system.
  - Build second module. Pick a module that can work with the previous module.
  - Decide if you want to build HTTP API or TUI for the next phase.

### Features

- [ ] Build core for abstracting modules.
- [ ] Build pomodoro module that works with the task module.
- [ ] Plan and document your module system.
