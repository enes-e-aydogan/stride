# Project Goals

## Primary Goal

Build a modular productivity system that demonstrates architectural thinking for my portfolio.

## What I Want to Build

- Multiple modules that can work independently and together.
- Modules that can be enabled or disabled via configuration at startup or runtime.
- A core that can be extended with CLI, TUI, or web interfaces without changing module logic.
- An HTTP API for integration with other services, with versioning and clear contracts.
- Basic CI and automated tests for core lifecycle and at least one module.

## What I want to learn

- Technical
  - Go programming language idioms and best practices
  - SQL and schema migrations
  - Building abstractions and clean architecture
  - Docker containerization
  - Deploying and managing server-side services
  - Writing and understanding how HTTP works
  - Designing simple, user-friendly APIs and UIs

- Process & quality
  - Test-driven development (TDD) and writing effective unit/integration tests
  - How to write good project documentation (README, contributing guide, API docs)
  - Project planning and breaking work into milestones, tasks, and acceptance criteria
  - Writing clean, maintainable code

## Success Criteria

- At least 3 working modules
- Plugin system
- CLI interface working end-to-end
- SQLite persistence across modules
- Deployed and accessible online
- Case study written explaining key decisions
- HTTP API
- Simple UI. TUI or web-based.
- Comprehensive test coverage
